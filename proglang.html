Networking & The Internet Terminology

TERM DEFINITION
Machine Language A encoding system that consist of sequences of instructions that are encoded as numeric digits.
Debugging The process of locating and correcting errors found with a machine language.
Assembler Programs used to convert mnemonic expressions into machine language instructions.
Assembly Language A mnemonic system for representing programs.
Translator A program that translated programs expressed high-level primitives into machine-language programs.
Compiler A translation program that compiled several machine instructions into short sequences to simulate the activity requested by a single high-level primitive.
Interpreter Programs that execute instructions as they were translated instead of recording the translated version for future use.
Cross-platform software Software that is independent of an operating system's design as well as the machine's hardware design and is therefore executable throughout a network.
Natural Language Language that evolved over time without formal grammatical analysis (ie English, German, and Latin)
Formal Language Language that is precisely defined by grammars (ie programming languages)
Machine independence The ability for a third-generation language to be compiled as easily for one machine as for another as the language used does not refer to the attributes of any particular machine.
High-level Programming Language Provide more abstractions for the programmer and make it easier for people to read and write a program.
Object-oriented programming (OOP) A programming paradigm in which a software system is viewed as a collection of units, called objects, each of which is capable of performing the actions that are immediately related to itself as well as requesting actions of other objects.
Scripting language The collection of languages that are a subset of the imperative programming languages.
