Operating Systems Terminology

TERM DEFINITION
Operating System The software that controls the overall operation of a computer including file management, the user interface, and the program execution environment.
Job queue Where jobs residing in mass storage wait for execution, in batch processing systems.
FIFO Stands for first-in, first-out and is the fashion in which objects in a queue are executed.
Interactive Processing Operating systems that allow a program being executed to carry on a dialogue with the user through remote terminals.
Time-Sharing Operating systems that provide service to multiple users at the same time.
Multiprogramming A technique in which time is divided into intervals and then the execution of each job is restricted to only one interval at a time, and can be used as a means of implementing time-sharing.
Application Software Consists of the programs for performing tasks particular to the machine's utilization, examples include, spreadsheets, database systems, desktop publishing systems, accounting systems, program development software, and games.
System Software Performs those tasks that are common to computer systems in general. Essentially provides the infrastructure that the application software requires.
Utility Software A category within the class of system software that consists of software units.
Graphical User Interface (GUI) performs tasks in which objects to be manipulated, such as files and programs, are represented pictorially on the display as icons.
Shell Older user interfaces that communicated with users through textual messages using a keyboard and monitor screen.
Kernel The internal part of an operating system. It contains those software components that perform the very basic functions required by the computer installation.
Device Driver A componenet of the kernel which are the software units that communicate with the controllers (or at times, directly with peripheral devices) to carry out operations on the peripheral devices attached to the machine. It translates generic requests into the more technical steps required by the device assigned to that driver.
File Manager A software component or unit within the operating system's kernel whose job is to coordinate the use of the machine's mass storage facilities.
Memory Manager A component of an operating system's kernel which is charged with the task of coordinating the machine's use of main memory.
Virtual Memory A Large "fictional" memory space created by paging.
Boot Strapping A procedure that is performed by a computer each time it is turned on that transfers the operating system from mass storage (where it is permanently stored) into main memory (which is essentially empty when the machine is first turned on).
ROM Stands for read-only memory and is a portion of a computer's main memory who's contents can be read but not altered.
Deadlock A problem that can arise during resource allocation and is the condition in which two or more processes are blocked from progressing because each is waiting for a resource that is allocated to another.
Spooling A technique of holding data for output at a later but more convenient time.
Super User (or Adminstrator) The person who establishes accounts and gains highly privileged access to the operating system, including altering settings, by identifying him- or herself as the administrator during the login procedure.
Multi-factor Authentication The procedure in which people gain access to accounts by more than just the use of a password, additional authentification factors can include security tokens, special access codes, fingerprints, voice, or retinal pattern.
